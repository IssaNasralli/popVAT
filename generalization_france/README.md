# Cross-Country Generalization on France (Île-de-France)

This repository includes a cross-country validation experiment demonstrating that the  **popVAT** model generalizes beyond Tunisia.

The experiment was conducted on **Île-de-France**, the most densely populated metropolitan region in France, to assess whether the proposed architecture remains effective under different geographic, demographic, and urban conditions.

> **Objective**  
> The goal of this experiment is ** to train a France-specific model**,  then to evaluate the popVAT architecture using exactly the same methodology and configuration developed for Tunisia.

## Experimental Setup

The French experiment follows exactly the same framework described in the paper:

- Identical preprocessing workflow
- Same raster alignment strategy
- Logarithmic transformation of population targets
- Unchanged neural network architecture and hyperparameters
- Same training and inference procedures

The supervisory reference layer is the **WorldPop 2020 population dataset** at **100 m spatial resolution**.

Ancillary datasets are the same as those used for Tunisia, with one exception:

- Educational Points of Interest (POIs) were obtained from the French Open Data Platform for Education, Sports and Youth.

Administrative-level evaluation was performed using official 2020 population statistics from **INSEE** (Institut national de la statistique et des études économiques).

---

## Performance on Île-de-France

The pretrained popVAT model achieved an excellent coefficient of determination:

\[
R² = 0.9636
\]

This result confirms that popVAT preserves strong predictive performance in a completely different national context, highlighting the model's ability to generalize across countries without any architectural modification.

# Model Evaluation of popVAT

This folder contains the scripts used to evaluate predicted population maps.

The evaluation follows the standard approach used in population-mapping studies: pixel-level population predictions are aggregated to administrative units and compared with official census data.

---

## Files in This Folder

- `data_atrous.py`  
  Utilities for loading raster data, administrative masks, and census tables.

- `evaluate.py`  
  Functions used to aggregate predictions and compute evaluation metrics.

- `France_Regions/`  
  This folder must be present for the evaluation to run. It contains **8 raster masks**, each representing one administrative subdivision used in the experiment. These masks are used to aggregate pixel-level population predictions into region-level population totals.

- `insee_population.csv`  
  Official 2020 population statistics from INSEE.

---

## High-Level Evaluation Pipeline

The evaluation process follows these steps:

1. Load the predicted population raster.
2. Load administrative region masks.
3. Load official population statistics.
4. Aggregate predicted population within each region.
5. Compare predicted totals with census data.
6. Compute statistical indicators.

This procedure measures how accurately the model reproduces official population distributions across administrative regions.

---

## Running the Evaluation

```bash
python evaluate.py \
  --raster france10_popVAE_full_GAG_1025_20__best_weights_popVAE_full_France_GAG_1025_20_21.h5__21_predicted.tif \
  --csv insee_population.csv \
  --region France \
  --nb_masks 8
```

### Parameters

| Parameter | Description |
|----------|-------------|
| `--raster` | Path to the predicted population raster |
| `--csv` | CSV file containing official INSEE population statistics |
| `--region` | Country or region name (`France`) |
| `--nb_masks` | Number of administrative masks (`8`) |

---

## Expected Output

The evaluation script reports administrative-level comparison metrics, including:

- Coefficient of determination (`R²`)
- Predicted and official population totals by region

For the provided Île-de-France experiment, the expected result is:

```text
R² = 0.9636
```
